## Description

Configure Tailwind CSS for the frontend project. Tailwind provides utility-first CSS that enables rapid UI development with consistent design tokens.

---

## Acceptance Criteria

- [x] Tailwind CSS installed and configured
- [x] PostCSS configured correctly
- [x] Global styles use Tailwind directives
- [x] Test page renders with Tailwind styles
- [x] Dark mode support configured
- [x] Production build works with Tailwind purge

---

## Files to Create/Modify

| File | Purpose |
|------|---------|
| `frontend/postcss.config.mjs` | PostCSS configuration with @tailwindcss/postcss |
| `frontend/app/globals.css` | Tailwind import (`@import "tailwindcss"`) |
| `frontend/app/page.tsx` | Update with Tailwind classes |

Note: Tailwind v4 uses CSS-first configuration, no `tailwind.config.ts` needed for basic setup.

---

## Implementation Steps

### Step 1: Install Tailwind CSS
1. Install tailwindcss, postcss, autoprefixer

### Step 2: Initialize Configuration
1. Run `npx tailwindcss init -p --ts`
2. Configure content paths

### Step 3: Add Tailwind Directives
1. Update globals.css with @tailwind directives

### Step 4: Update Components
1. Convert page.tsx to use Tailwind classes

### Step 5: Verify Build
1. Run development server
2. Run production build
